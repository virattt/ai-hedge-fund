# 网络重试机制修复说明

## 问题描述

运行程序时遇到网络连接错误：

```
requests.exceptions.ConnectionError: ('Connection aborted.',
RemoteDisconnected('Remote end closed connection without response'))
```

### 错误原因

1. **远程服务器关闭连接**：金融数据 API 服务器在请求过程中意外关闭了连接
2. **缺少连接错误重试**：原代码只处理了 HTTP 429（限流）错误，没有处理网络连接异常

### 触发场景

- 网络不稳定
- API 服务器负载高
- 请求超时
- 连接被重置

## 修复方案

在 `src/tools/api.py` 的 `_make_api_request()` 函数中添加了完整的异常处理和重试机制。

### 修改内容

#### 1. 添加超时设置

```python
# 修改前
response = requests.post(url, headers=headers, json=json_data)
response = requests.get(url, headers=headers)

# 修改后
response = requests.post(url, headers=headers, json=json_data, timeout=30)
response = requests.get(url, headers=headers, timeout=30)
```

#### 2. 添加异常捕获和重试

```python
try:
    # 执行请求
    if method.upper() == "POST":
        response = requests.post(url, headers=headers, json=json_data, timeout=30)
    else:
        response = requests.get(url, headers=headers, timeout=30)

    # 处理 429 限流
    if response.status_code == 429 and attempt < max_retries:
        delay = 60 + (30 * attempt)
        print(f"Rate limited (429). Retrying in {delay}s...")
        time.sleep(delay)
        continue

    return response

except (requests.exceptions.ConnectionError,
        requests.exceptions.Timeout,
        requests.exceptions.ChunkedEncodingError) as e:
    # 处理连接错误
    if attempt < max_retries:
        delay = 2 ** (attempt + 1)  # 指数退避：2s, 4s, 8s
        print(f"Connection error: {type(e).__name__}. Retrying in {delay}s...")
        time.sleep(delay)
        continue
    else:
        print(f"Connection error after {max_retries + 1} attempts")
        raise
```

### 重试策略

#### 限流错误（429）
- **策略**：线性退避
- **延迟**：60s, 90s, 120s, 150s...
- **原因**：API 限流需要较长等待时间

#### 连接错误
- **策略**：指数退避
- **延迟**：2s, 4s, 8s
- **原因**：网络问题通常短暂，快速重试更合理

### 捕获的异常类型

1. **ConnectionError**：连接被拒绝或重置
2. **Timeout**：请求超时（30秒）
3. **ChunkedEncodingError**：数据传输中断

## 额外改进

### 1. 请求超时
添加了 30 秒超时，避免请求无限期挂起

### 2. 更好的错误信息
```python
print(f"Connection error: {type(e).__name__}. Attempt {attempt + 1}/{max_retries + 1}")
```

### 3. 异常传播
最后一次重试失败后，正确抛出异常，让上层代码处理

## 最佳实践
