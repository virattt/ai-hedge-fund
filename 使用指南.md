# AI 对冲基金 - 快速使用指南

## 问题总结

项目的 README 中的命令示例**不完整**，会导致程序在非交互式环境中失败。原因是程序需要交互式终端输入来选择分析师和模型。

## 正确的使用方法

### 1. 确认依赖已安装

```bash
poetry install
```

### 2. 检查 API 密钥配置

确保 `.env` 文件中配置了 API 密钥：

```bash
# 美团 AIGC 平台（OpenAI 兼容）
OPENAI_API_KEY=your-key-here
OPENAI_API_BASE=https://aigc.sankuai.com/v1/openai/native

# 金融数据 API（可选，AAPL/GOOGL/MSFT/NVDA/TSLA 免费）
FINANCIAL_DATASETS_API_KEY=your-key-here
```

### 3. 运行对冲基金系统

**关键：必须指定 `--analysts-all` 和 `--model` 参数！**

```bash
# 基本示例
poetry run python src/main.py \
  --tickers AAPL \
  --analysts-all \
  --model "MiniMax-M2.5" \
  --start-date 2026-02-28 \
  --end-date 2026-03-01

# 多个股票
poetry run python src/main.py \
  --tickers AAPL,MSFT,NVDA \
  --analysts-all \
  --model "MiniMax-M2.5"

# 显示推理过程
poetry run python src/main.py \
  --tickers AAPL \
  --analysts-all \
  --model "MiniMax-M2.5" \
  --show-reasoning

# 指定特定分析师
poetry run python src/main.py \
  --tickers AAPL \
  --analysts "warren_buffett,charlie_munger,michael_burry" \
  --model "MiniMax-M2.5"
```

### 4. 运行回测系统

```bash
# 基本回测
poetry run python src/backtester.py \
  --tickers AAPL,MSFT,NVDA \
  --analysts-all \
  --model "MiniMax-M2.5"

# 指定日期范围
poetry run python src/backtester.py \
  --tickers AAPL \
  --analysts-all \
  --model "MiniMax-M2.5" \
  --start-date 2026-01-01 \
  --end-date 2026-03-01

# 使用备用命令
poetry run backtester \
  --tickers AAPL \
  --analysts-all \
  --model "MiniMax-M2.5"
```

## 常用参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--tickers` | 股票代码（必需） | `AAPL,MSFT,NVDA` |
| `--analysts-all` | 使用所有分析师 | 无值 |
| `--analysts` | 指定分析师 | `warren_buffett,charlie_munger` |
| `--model` | 模型名称（必需） | `MiniMax-M2.5` |
| `--start-date` | 开始日期 | `2026-01-01` |
| `--end-date` | 结束日期 | `2026-03-01` |
| `--initial-cash` | 初始资金 | `100000.0` |
| `--margin-requirement` | 保证金比率 | `0.5` |
| `--show-reasoning` | 显示推理 | 无值 |
| `--show-agent-graph` | 显示代理图 | 无值 |
| `--ollama` | 使用本地 Ollama | 无值 |

## 已知问题

1. **交互式输入问题**：如果不指定 `--analysts-all`/`--analysts` 和 `--model`，程序会尝试启动交互式选择界面，在非交互式环境中会失败。

2. **代码 Bug**：`src/agents/news_sentiment.py` 第 132 行存在变量未初始化的问题：
   ```
   UnboundLocalError: cannot access local variable 'sentiments_classified_by_llm'
   ```
   这个问题在某些条件下会导致程序崩溃。

## 技巧

1. **快速测试**：使用单个股票和短日期范围进行快速测试
   ```bash
   poetry run python src/main.py \
     --tickers AAPL \
     --analysts-all \
     --model "MiniMax-M2.5" \
     --start-date 2026-02-28 \
     --end-date 2026-03-01
   ```

2. **调试模式**：添加 `--show-reasoning` 查看每个代理的决策过程

3. **使用本地模型**：如果安装了 Ollama，可以使用 `--ollama` 参数避免 API 调用费用

4. **选择特定分析师**：如果只想测试某些投资策略，使用 `--analysts` 而不是 `--analysts-all`
