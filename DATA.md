## Data Pipeline for Backtesting the AI Hedge Fund

This document explains how we pull and cache historical data so we can backtest the AI Hedge Fund logic and run autoresearch safely and reproducibly.

### 1. Data Sources and Responsibilities

- **Price data (equities, sectors, tastytrade sleeve, Hyperliquid sleeve)**: pulled via `src/tools/api.get_prices` (e.g. Financial Datasets / Yahoo). Cached as JSON under `autoresearch/cache/`.
- **Analyst signals (18-agent committee)**: generated by `src/main.run_hedge_fund` and cached by `autoresearch/cache_signals.py` as `signals.json` for fast, LLM-free backtests.
- **Hedge-fund backtester & autoresearch**: consume the cached price / signal files via `autoresearch/evaluate.py`, `autoresearch/fast_backtest.py`, and `autoresearch/portfolio_backtest.py`.
- **Fundamental / macro / crypto data (optional, higher cost)**: designed to come from [Financial Datasets](https://www.financialdatasets.ai/) via `src/tools/api` wrappers and cached to JSON once, then reused across backtests.

> **Key idea:** expensive calls (LLMs, external APIs) happen once during caching; backtests and autoresearch then run purely on local JSON files.

---

### 2. Core Script: `autoresearch/cache_signals.py`

Entry point:

```bash
poetry run python -m autoresearch.cache_signals [options]
```

Main behaviors:

- **Phase 1 (always): cache prices**
  - Calls `get_prices(ticker, start, end)` for each ticker.
  - Writes OHLCV history to `autoresearch/cache/<PRICES_PATH>.json`.
  - Backs up the previous file as `<PRICES_PATH>.json.bak`.
- **Phase 2 (optional): cache analyst signals**
  - If `--prices-only` is omitted, runs the full 18-agent pipeline (`run_hedge_fund`) for each business day and saves a daily snapshot of analyst signals into `autoresearch/cache/signals.json`.
  - Resumes from partial progress (already-cached dates are skipped).

CLI options (subset; see `cache_signals.py` for full list):

- `--tickers A,B,C` — comma-separated tickers.
- `--start YYYY-MM-DD` — backtest start date (also used for price cache).
- `--end YYYY-MM-DD` — backtest end date.
- `--prices-only` — **only** cache prices, skip LLM signals.
- `--prices-path prices_<name>.json` — output filename under `autoresearch/cache/`.
- `--model`, `--provider` — model + provider for full signal caching (Mode 2).

> **Rule of thumb:** match `--start` / `--end` to the `BACKTEST_START` / `BACKTEST_END` in `autoresearch/params*.py` so caches fully cover your backtest window.

### 3. Prices-Only Mode (Recommended Default)

Prices-only caching is free (no LLM calls) and is what most strategy and sector tuning uses.

#### 3.1 Sector price caches

Run from repo root:

```bash
# Memory
poetry run python -m autoresearch.cache_signals \
  --tickers MU,WDC,STX \
  --prices-only \
  --prices-path prices_memory.json

# Equipment
poetry run python -m autoresearch.cache_signals \
  --tickers AMAT,ASML,LRCX,KLAC,TEL \
  --prices-only \
  --prices-path prices_equipment.json

# Networking
poetry run python -m autoresearch.cache_signals \
  --tickers ANET,AVGO,MRVL \
  --prices-only \
  --prices-path prices_networking.json

# Power infra
poetry run python -m autoresearch.cache_signals \
  --tickers VRT,CEG,EQT \
  --prices-only \
  --prices-path prices_power_infra.json
```

These JSON files live in `autoresearch/cache/` and are gitignored. `RUNBOOK.md` has the full sector list and a `refresh_all_prices.sh` helper to update everything in one shot.

#### 3.2 Tastytrade sleeve (Dexter) tickers

To cache prices for the **tastytrade AI infra sleeve** Dexter defines, you can either:

- **Single shared cache** for all sleeve names, or
- **Per-group caches** (e.g. core vs builders vs satellites).

Example: one shared cache for the tastytrade sleeve:

```bash
poetry run python -m autoresearch.cache_signals \
  --tickers ASML,AMAT,KLAC,LRCX,SNPS,CDNS,ANET,VRT,AVGO,MRVL,CEG,EQT,WDC,STX,LITE \
  --start 2024-01-01 \
  --end 2026-03-07 \
  --prices-only \
  --prices-path prices_tastytrade_sleeve.json
```

You can then point `autoresearch/evaluate.py` or `portfolio_backtest.py` at this file via `--prices-path` when backtesting strategies that specifically target the tastytrade sleeve.

### 4. Full-Signal Mode (18-Agent Committee Cache)

Full-signal mode precomputes all 18 agents’ daily signals, enabling:

- Analyst trust weight tuning (which agents add alpha vs noise).
- Committee-driven backtests that use real per-agent signals instead of only technicals.

This is **expensive** (LLM calls per business day), so run it sparingly:

```bash
poetry run python -m autoresearch.cache_signals \
  --tickers AAPL,NVDA,MSFT,GOOGL,TSLA \
  --start 2025-01-02 \
  --end 2026-03-07
```

Requirements and behavior:

- Requires an LLM key (e.g. `OPENROUTER_API_KEY`, see `README.md`).
- Uses `model` / `provider` defaults from `cache_signals.py` (currently `z-ai/glm-4.5-air` via OpenRouter).
- Writes/updates `autoresearch/cache/signals.json` and `autoresearch/cache/meta.json`.
- Automatically resumes on restart; already-cached dates are skipped.

> **Best practice:** start with prices-only Mode 1 for strategy/autoresearch tuning; only add full-signal Mode 2 once the technical harness is solid and you specifically want to tune analyst weights.

### 5. How Backtests Consume Cached Data

#### 5.1 Single-sector backtests

```bash
# Use sector-specific params and an appropriate price cache
poetry run python -m autoresearch.evaluate \
  --params autoresearch.params_equipment \
  --tickers AMAT,ASML,LRCX,KLAC,TEL \
  --prices-path prices_equipment.json
```

Key relationships:

- `params_<sector>.py` defines the backtest window and strategy parameters.
- `--tickers` and `--prices-path` must match a cached JSON file that covers the full date range.
- If `signals.json` exists (full-signal mode), `evaluate` can also pull per-agent signals; otherwise it falls back to technical-only mode.

#### 5.2 Portfolio backtests (multi-sector)

```bash
poetry run python -m autoresearch.portfolio_backtest --weights oos
```

Under the hood:

- Loads per-sector returns using their respective `prices_<sector>.json` caches.
- Combines them using the chosen weighting scheme (`equal`, `sharpe`, `oos`, etc.).
- Assumes each sector cache is up to date; use `refresh_all_prices.sh` or rerun `cache_signals` for any stale sector.

### 6. Financial Datasets Integration (Equities, Crypto, Macro)

We use [Financial Datasets](https://www.financialdatasets.ai/) as the primary paid data source behind `get_prices` and future fundamental / macro features. Their API groups endpoints into **Stocks**, **Crypto**, and **Macroeconomics**:

- **Stocks (examples)**  
  - `GET /prices` — stock OHLCV (currently used by `get_prices`)  
  - `GET /company/facts` — static company facts  
  - `GET /earnings` — earnings history  
  - `GET /financial-metrics` — time-series metrics (margins, leverage, growth, etc.)  
  - `GET /financials`, `/financials/balance-sheets`, `/financials/cash-flow-statements`, `/financials/income-statements`, `/financials/segmented-revenue` — full financial statements by period  
  - `GET /analyst-estimates` — consensus estimates and revisions  
  - `GET /insider-trades` — insider buying/selling  
  - `GET /institutional-ownership` — institutional holdings  
  - `GET /news` — news articles  
  - `GET /filings` — SEC filings
- **Crypto**
  - `GET /crypto/prices` — OHLCV for crypto assets.
- **Macroeconomics**
  - `GET /macro/interest-rates` — interest-rate histories for macro regime detection.

Their pricing is per-request (e.g. `/prices` ≈ $0.01, `/financials` ≈ $0.10), but they explicitly allow us to **save responses and avoid repeat calls**, which is exactly what our cache layer is for.

#### 6.1 Pulling more price history (depth)

Use `cache_signals.py` with a longer `--start` for each universe:

```bash
# Example: tastytrade AI infra sleeve, deep history (2018+)
poetry run python -m autoresearch.cache_signals \
  --tickers ASML,AMAT,KLAC,LRCX,SNPS,CDNS,ANET,VRT,AVGO,MRVL,CEG,EQT,WDC,STX,LITE \
  --start 2018-01-01 \
  --end 2026-03-07 \
  --prices-only \
  --prices-path prices_tastytrade_sleeve_long.json
```

This triggers one `/prices` call per ticker through `get_prices`, at ≈$0.01 each, and writes a long-history JSON cache that backtests and autoresearch can use to:

- Test strategies across multiple regimes (pre-2020, COVID crash, 2022 hiking cycle, 2023–25 AI melt-up).
- Run walk-forward / rolling-window validation instead of a single 2025–26 slice.

#### 6.2 Adding more tickers (breadth)

To broaden the universe (e.g. challengers Dexter excluded, full sector cohorts, or additional regions), call the same flow with a larger ticker list:

```bash
poetry run python -m autoresearch.cache_signals \
  --tickers AMAT,ASML,LRCX,KLAC,TEL,NVDA,AVGO,ARM,AAPL,COHR,CIEN,BE,SEI,CRWV,CORZ \
  --start 2018-01-01 \
  --end 2026-03-07 \
  --prices-only \
  --prices-path prices_ai_infra_and_challengers.json
```

More breadth improves:

- Cross-sectional robustness (does a rule work across many names, or just the poster children?).
- Sector/industry correlation estimates and portfolio construction.

#### 6.3 Fundamentals, events, and macro (dimensions)

We can mirror `get_prices` with small wrappers in `src/tools/api.py`, e.g.:

- `get_financials(ticker, start, end)` → `/financials`
- `get_financial_metrics(ticker, start, end)` → `/financial-metrics`
- `get_earnings(ticker)` → `/earnings`
- `get_insider_trades(ticker, start, end)` → `/insider-trades`
- `get_institutional_ownership(ticker, date)` → `/institutional-ownership`
- `get_news(ticker, start, end)` → `/news`
- `get_interest_rates(start, end)` → `/macro/interest-rates`
- `get_crypto_prices(symbol, start, end)` → `/crypto/prices`

Each wrapper should:

1. Take a ticker (or symbol) and a date window.
2. Call the appropriate Financial Datasets endpoint.
3. Dump the structured JSON under `autoresearch/cache/`, e.g.:
   - `financials_<universe>.json`
   - `financial_metrics_<universe>.json`
   - `earnings_<universe>.json`
   - `insider_trades_<universe>.json`
   - `institutional_ownership_<universe>.json`
   - `news_<universe>.json`
   - `macro_rates.json`
   - `crypto_prices_<universe>.json`

##### 6.3.1 Per-ticker “pull everything” recipe

For **one ticker** (e.g. `NVDA`) we want a single, repeatable recipe that pulls **all** supported dimensions and stores them under `autoresearch/cache/`:

1. **Prices** (if not already cached via `cache_signals`):

   ```bash
   poetry run python -m autoresearch.cache_signals \
     --tickers NVDA \
     --start 2018-01-01 \
     --end 2026-03-07 \
     --prices-only \
     --prices-path prices_hl_nvda.json
   ```

2. **Financial statements + metrics** (via new wrappers):

   ```bash
   # pseudo-CLI we will wire up via a small script, e.g. autoresearch/cache_fundamentals.py
   poetry run python -m autoresearch.cache_fundamentals \
     --tickers NVDA \
     --start 2018-01-01 \
     --end 2026-03-07 \
     --output-prefix nvda
   ```

   Expected outputs:

   - `autoresearch/cache/financials_nvda.json`
   - `autoresearch/cache/financial_metrics_nvda.json`
   - `autoresearch/cache/earnings_nvda.json`

3. **Insider, institutional, news**:

   ```bash
   poetry run python -m autoresearch.cache_events \
     --tickers NVDA \
     --start 2018-01-01 \
     --end 2026-03-07 \
     --output-prefix nvda
   ```

   Expected outputs:

   - `autoresearch/cache/insider_trades_nvda.json`
   - `autoresearch/cache/institutional_ownership_nvda.json`
   - `autoresearch/cache/news_nvda.json`

4. **Macro + crypto (shared across tickers)**:

   ```bash
   poetry run python -m autoresearch.cache_macro \
     --start 2018-01-01 \
     --end 2026-03-07

   poetry run python -m autoresearch.cache_crypto \
     --symbols BTC,ETH \
     --start 2018-01-01 \
     --end 2026-03-07 \
     --output-prefix core_crypto
   ```

   Outputs:

   - `autoresearch/cache/macro_rates.json`
   - `autoresearch/cache/crypto_prices_core_crypto.json`

> **Design note:** `cache_fundamentals.py`, `cache_events.py`, `cache_macro.py`, and `cache_crypto.py` are intentionally symmetric with `cache_signals.py`: loop over tickers/dates, call a single API wrapper per endpoint, and write JSON files that downstream backtests and agents can read without ever hitting the external API again.

Once cached, these become inputs to:

- **Factor signals**: valuation, quality, growth, profitability built from statements/metrics.
- **Event-aware risk**: earnings windows, insider dumps, big ownership shifts, negative news.
- **Macro overlays**: rate regimes (hiking vs cutting) influencing exposure and position sizing.
- **Crypto sleeves**: historical BTC/ETH/etc. returns for Hyperliquid-style overlays or hedges.

#### 6.4 Why burning credits is justified

Think of Financial Datasets spend as **one-time data capex**:

- `/prices` at ~$0.01 per ticker over a multi-year window:
  - 200 tickers × $0.01 ≈ **$2** for deep history on the entire AI infra + challenger universe.
- Statements/metrics/endpoints are more expensive but:
  - Pulled once per period per ticker, then cached and reused.
  - Unlock much richer, economically-grounded backtests (not just price wiggles).

Benefits to backtesting:

- **More regimes** → less overfitting to 2025–26.
- **More names** → cross-sectional robustness and better portfolio statistics.
- **More dimensions** (fundamentals, events, macro) → realistic signals that align with how the 18 agents claim to reason.

Net: a small, controlled credit burn up front yields a durable local dataset that every future backtest, autoresearch session, and Dexter sleeve evaluation can reuse at zero marginal cost.

### 9. Operational Checklist Before a Backtest

1. **Confirm cache coverage**: `prices_*.json` date range fully covers `BACKTEST_START` → `BACKTEST_END`.
2. **Refresh stale prices**: run `./autoresearch/refresh_all_prices.sh` or the relevant `cache_signals --prices-only` commands.
3. **(Optional) Refresh signals**: if you rely on `signals.json`, re-run `cache_signals` for the desired window.
4. **Lock params**: ensure `autoresearch/params*.py` reflect the window and strategy you want to test.
5. **Run**:
   - Sector evals via `autoresearch.evaluate`.
   - Portfolio eval via `autoresearch.portfolio_backtest`.
   - Autoresearch loops via `autoresearch.run_autoresearch_loop` once the data layer is green.
