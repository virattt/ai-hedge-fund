# 位置: src/agents/portfolio_manager.py

它的核心作用是把上游分析结果变成每个股票的最终交易动作（buy/sell/short/cover/hold）。

读取输入状态
来自 state["data"] 的组合数据、分析师信号、ticker 列表。见 src/agents/portfolio_manager.py

对接风险管理结果
从 risk manager 输出里拿每个 ticker 的 remaining_position_limit 和 current_price，算出 max_shares，确保不能超风险上限。见 src/agents/portfolio_manager.py

汇总多分析师信号
把不同 agent 的信号压缩成 {sig, conf} 结构，减少提示词体积。见 src/agents/portfolio_manager.py

生成可执行约束
通过 compute_allowed_actions 先做“硬约束”计算：哪些动作允许、每个动作最大数量是多少。见 src/agents/portfolio_manager.py

调用 LLM 做最终选择
在允许动作集合里，让 LLM 选每个 ticker 的一个动作和数量，并给置信度与简短理由。见 src/agents/portfolio_manager.py

回写输出到图状态
输出一条消息（JSON 决策）并把更新后的 data 返回给下游。见 src/agents/portfolio_manager.py

它解决的问题：

把“多分析师意见 + 风险约束”统一成可执行交易指令
防止 LLM 产生越权交易（先算 allowed actions，再让 LLM 选）
在无交易空间时自动降级为 hold，保证系统稳健运行


LLM 可选动作一共有 5 个：

buy
sell
short
cover
hold
定义在 src/agents/portfolio_manager.py#L13 的 PortfolioDecision.action 枚举里。

但每个 ticker 实际给 LLM 的是“子集”，由规则函数动态算出来：

计算逻辑在 src/agents/portfolio_manager.py#L95
传给 LLM 的 allowed 集合在 src/agents/portfolio_manager.py#L212
关键点：

hold 始终可选
buy 只有在有现金且价格有效时才会出现
sell 只有已有多头仓位时才会出现
short 只有满足 max_qty 和保证金约束时才会出现
cover 只有已有空头仓位时才会出现
所以，LLM 不是在 5 个动作里随便选，而是在该 ticker 的 allowed 动作集合里选 1 个，并给对应数量。

# risk_manager.py
这个函数是风险控制节点，核心作用是“把分析信号变成可执行的风险上限”。

函数位置: src/agents/risk_manager.py

输入是什么

state: AgentState
定义见 src/graph/state.py，主要包含三块：
messages: 上游节点消息历史
data: 业务数据（这个函数实际用到）
metadata: 配置（比如是否打印推理）
agent_id: str（默认 "risk_management_agent"）
用于进度日志和写回信号键名。
这个函数从 state["data"] 中重点读取：

portfolio（现金、持仓）
tickers（本次需要评估的股票）
start_date、end_date（取价区间）
另外从 state 中取 API key 去拉价格数据
输出是什么
返回一个字典，结构在 src/agents/risk_manager.py：

messages
把当前风险分析结果（JSON字符串）追加成一条 HumanMessage。

data
返回更新后的 data，其中最关键是写入：

state["data"]["analyst_signals"][agent_id] = risk_analysis
也就是把每个 ticker 的风险约束结果交给下游组合经理使用。
risk_analysis 每个 ticker 里包含：

remaining_position_limit（还能开多大仓位，美元）
current_price
volatility_metrics（日波动、年化波动、分位数）
correlation_metrics（与已持仓相关性）
reasoning（组合价值、当前暴露、调整因子等解释）
它解决了什么问题
它解决的是“下单前的仓位风控”问题，避免组合经理只看看涨信号就过度下注。

具体做法：

先算每个标的波动率
波动越大，允许仓位比例越小。

再算与现有持仓的相关性
相关性越高，进一步降低仓位，防止集中风险。

结合组合净值与现金约束
得到最终可用的 remaining_position_limit，给下游决策作为硬约束。

一句话总结：这个函数把“能不能买、最多买多少”量化出来，防止策略在高波动或高相关场景下失控。
